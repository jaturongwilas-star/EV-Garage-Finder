# EV Garage Finder

เว็บแอปค้นหาอู่ซ่อมรถยนต์ไฟฟ้า (EV) ทั่วประเทศไทย — ค้นหาด้วยคำค้น กรองตามจังหวัด ยี่ห้อรถ และงานที่รับซ่อม เป็นเว็บ static site ฝั่งหน้าบ้าน ส่วนฐานข้อมูลจริงอยู่บน Google Sheet ผ่าน Google Apps Script (ดูหัวข้อ "การตั้งค่า Backend" ด้านล่าง)

## ⚠️ ก่อน push ขึ้น GitHub — เช็คโครงสร้างไฟล์ให้ตรงเป๊ะ

ปัญหาที่เจอบ่อยคือมีไฟล์ซ้ำ/ไฟล์เก่าหลงเหลืออยู่คนละที่กับที่เว็บอ้างอิงจริง (เช่น `js/app.js` เป็นโค้ดเก่า แต่มีไฟล์ `app.js` เวอร์ชันใหม่ไปอยู่ที่ root แทน — เว็บจะยังโหลดโค้ดเก่าอยู่เพราะ `index.html` เรียก `js/app.js` ตรงๆ) ก่อน push ให้ตรวจสอบว่าโครงสร้างไฟล์ตรงกับหัวข้อ "โครงสร้างไฟล์" ด้านล่างเป๊ะๆ — **ไม่ควรมีไฟล์ `app.js`, `style.css`, `database.json`, หรือ `Code.gs` ลอยอยู่ที่ root ของ repo เด็ดขาด** ไฟล์เหล่านี้ต้องอยู่ใน `js/`, `css/`, `data/`, `apps-script/` ตามลำดับเท่านั้น ถ้าเจอไฟล์ซ้ำแบบนี้ให้ลบไฟล์ที่ root ทิ้ง แล้วเช็คว่าไฟล์ในโฟลเดอร์ย่อยเป็นเวอร์ชันล่าสุดจริง (เทียบกับแพ็กเกจนี้)

## ⚠️ หากเปิดแล้วหน้าเว็บไม่มีสไตล์ / ค้นหาแล้วได้ 0 รายการ

อาการนี้ส่วนใหญ่เกิดจาก 2 กรณี:
1. **แตกไฟล์ zip ไม่สมบูรณ์** — path ในแถบที่อยู่เบราว์เซอร์จะโชว์ชื่อโฟลเดอร์ซ้ำกัน เช่น `.../EV-Garage-Finder/EV-Garage-Finder/index.html` ซึ่งแปลว่าโฟลเดอร์ `css/`, `js/`, `data/` อาจไม่ได้อยู่ระดับเดียวกับ `index.html` จริงๆ
2. **ยังไม่ได้ตั้งค่า `APPS_SCRIPT_URL`** ในไฟล์ `js/app.js` (ดูหัวข้อ "การตั้งค่า Backend") — ถ้าเว็บเชื่อมต่อ Apps Script ไม่ได้ ระบบจะ fallback ไปใช้ข้อมูลสำรองใน `js/data.js` แทน (14 รายการ) ซึ่งจะเห็นข้อมูลแค่ชุดตัวอย่าง ไม่ใช่ 0 รายการ ถ้าเห็น 0 รายการจริงๆ ให้เช็ค Console (F12) หาข้อความ error สีแดง

ตรวจสอบว่า:
1. แตกไฟล์ zip แบบ "Extract Here" ไม่ใช่เปิดดูข้างในไฟล์ zip โดยไม่แตกไฟล์จริง
2. ในโฟลเดอร์ที่มี `index.html` ต้องเห็นโฟลเดอร์ `css`, `js`, `data`, `apps-script` อยู่ **ระดับเดียวกัน** (sibling) ไม่ใช่ซ้อนอยู่ในโฟลเดอร์ลูกอีกที และไม่มีไฟล์ซ้ำลอยอยู่ที่ root ตามหัวข้อด้านบน
3. เปิด DevTools ของเบราว์เซอร์ (กด F12) → แท็บ Console หรือ Network → รีโหลดหน้า → ถ้าเห็นบรรทัดสีแดงแบบ 404 ของ `style.css` หรือ `app.js` แปลว่าโครงสร้างโฟลเดอร์ผิด ให้จัดใหม่ตามข้อ 2

## โครงสร้างไฟล์

```
EV-Garage-Finder/
│
├── index.html          หน้าเว็บหลัก (โครงสร้าง + UI)
├── css/
│   └── style.css        ธีมและสไตล์ทั้งหมด
├── js/
│   ├── app.js            ตรรกะ: โหลดข้อมูลจาก Google Sheet, ค้นหา/กรอง, render การ์ด
│   └── data.js            ข้อมูลสำรอง (fallback) ใช้เมื่อเชื่อมต่อ Apps Script ไม่ได้ (เช่นเปิดไฟล์ตรงๆ)
├── data/
│   └── database.json     ไฟล์อ้างอิงชุดข้อมูลเริ่มต้น (ใช้ตอน seed ข้อมูลลง Google Sheet ครั้งแรก — ไม่ได้ถูกเว็บ fetch โดยตรงอีกต่อไป)
├── apps-script/
│   └── Code.gs            โค้ด backend (Google Apps Script) ที่ผูกกับ Google Sheet — ฐานข้อมูลจริง อยู่ที่นี่
├── .gitattributes
└── README.md
```

> หมายเหตุ: ไฟล์ตัวอย่าง Excel/CSV Template และไฟล์ QR ตัวอย่างที่เคยอยู่ในโปรเจกต์ถูกลบออกแล้ว เพราะไม่ได้ถูกใช้จริง — ปุ่ม "ดาวน์โหลด Template Excel" สร้างไฟล์ `garage_template.xlsx` ขึ้นในเบราว์เซอร์เองแบบ dynamic ไม่ต้องพึ่งไฟล์แยกในโปรเจกต์

## วิธีเปิดใช้งาน

**วิธีที่ 1 — เปิดผ่านเว็บเซิร์ฟเวอร์ (แนะนำ)**

รันเซิร์ฟเวอร์เล็กๆ ในโฟลเดอร์นี้ (ไม่จำเป็นเท่าเดิม เพราะดึงข้อมูลจาก Google Apps Script ผ่าน HTTPS อยู่แล้ว แต่ยังช่วยให้ path ของ css/js ถูกต้องแน่นอน):

```bash
cd EV-Garage-Finder
python3 -m http.server 8080
# แล้วเปิด http://localhost:8080
```

**วิธีที่ 2 — เปิดไฟล์ `index.html` ตรงๆ**

ก็ใช้งานได้เช่นกัน ถ้าตั้งค่า `APPS_SCRIPT_URL` ใน `js/app.js` ไว้แล้ว จะ fetch ข้อมูลจาก Google Sheet ได้ปกติ (Apps Script Web App ไม่ติดปัญหา CORS/`file://` เหมือน fetch ไฟล์ในเครื่อง) ถ้ายังไม่ได้ตั้งค่าหรือเชื่อมต่อไม่ได้ จะ fallback ไปใช้ข้อมูลที่ฝังไว้ใน `js/data.js` แทนอัตโนมัติ

**วิธีที่ 3 — Deploy ขึ้น GitHub Pages (วิธีที่ใช้งานจริง)**

Push โฟลเดอร์นี้ขึ้น repo `jaturongwilas-star/EV-Garage-Finder-` แล้วเปิดใช้งาน GitHub Pages branch `main` — หน้าเว็บ (static) นี้ทำหน้าที่แค่แสดงผล ส่วนฐานข้อมูลจริงอยู่ที่ Google Sheet ผ่าน Apps Script Web App ทุกเครื่อง/ทุกเบราว์เซอร์ที่เข้าเว็บจะโหลดข้อมูลชุดล่าสุดจาก Google Sheet เสมอ (มี cache-busting กันข้อมูลค้าง) — **ไม่มีการใช้ `localStorage` เก็บฐานข้อมูล และไม่ต้อง commit ข้อมูลขึ้น GitHub อีกต่อไป**

## วิธีเพิ่ม/แก้ไขข้อมูลอู่

วิธีหลักที่แนะนำ คือผ่านหน้าเว็บ (ปุ่ม "จัดการข้อมูล (แอดมิน)" → อัปโหลด Excel) ตามหัวข้อ "การอัปโหลดข้อมูลผ่าน Excel" ด้านล่าง ซึ่งจะบันทึกขึ้น Google Sheet ให้อัตโนมัติ

หรือจะเปิด Google Sheet ที่ผูกกับ Apps Script (`apps-script/Code.gs`) แล้วแก้ข้อมูลในชีต "Garages" ตรงๆ ก็ได้เหมือนกัน (แถวจะขึ้นเว็บทันทีในการโหลดครั้งถัดไป ไม่ต้อง deploy ใหม่) — แต่ละคอลัมน์มีโครงสร้างตรงกับตารางต้นฉบับ:

| field ใน JSON | ตรงกับคอลัมน์ในตาราง |
|---|---|
| `id` | ลำดับ (รหัสอ้างอิง) |
| `province` | จังหวัด |
| `district` | อำเภอ/เขต |
| `name` | ชื่ออู่ |
| `evTypes` | ประเภทรถ EV (array) |
| `services` | งานที่รับซ่อม (array) |
| `contact` | ผู้ติดต่อ |
| `phone` | โทรศัพท์ |
| `line` | Line (เช่น `@handle` หรือ URL เต็ม) |
| `lineQr` | QR Code Line (ใส่ลิงก์รูปภาพ QR Code LINE จริงของร้าน — ถ้าเว้นว่าง ระบบจะสร้าง QR จากช่อง Line ให้อัตโนมัติ) |
| `facebook` | Facebook (URL) |
| `googleMaps` | Google Maps (URL) |
| `hours` | วัน-เวลาทำการ |
| `notes` | หมายเหตุ |

ตัวอย่างการเพิ่มรายการใหม่ 1 รายการ (โครงสร้าง JSON ที่เว็บเห็น — ถ้าแก้ในชีตตรงๆ ให้พิมพ์ `evTypes` และ `services` เป็นข้อความคั่นด้วยจุลภาค เช่น `BYD, NETA` ในเซลล์เดียว ระบบจะแปลงเป็น array ให้เองตอนส่งออกเป็น JSON):

```json
{
  "id": "EV-015",
  "province": "นครราชสีมา",
  "district": "โชคชัย",
  "name": "ชื่ออู่ของคุณ",
  "evTypes": ["BYD", "NETA"],
  "services": ["ตรวจเช็คทั่วไป (PM)", "ระบบปรับอากาศ (HVAC)"],
  "contact": "ชื่อผู้ติดต่อ",
  "phone": "08x-xxx-xxxx",
  "line": "@yourline",
  "facebook": "https://facebook.com/yourpage",
  "googleMaps": "https://maps.google.com/?q=...",
  "hours": "จันทร์-เสาร์ 08:30-18:00",
  "notes": "หมายเหตุเพิ่มเติม (ใส่หรือปล่อยว่างก็ได้)"
}
```

ไฟล์ `data/database.json` ยังคงอยู่ในโปรเจกต์นี้เป็นแค่ **ข้อมูลอ้างอิงสำหรับ seed ลง Google Sheet ครั้งแรกเท่านั้น** (ฝังอยู่ในตัวแปร `SEED_DATA_JSON` ใน `apps-script/Code.gs` แล้ว ไม่ต้องอ่านไฟล์นี้อีก) หลังจากตั้งระบบเสร็จ ให้ถือว่า Google Sheet คือฐานข้อมูลจริงเพียงที่เดียว ไม่ต้องแก้ไฟล์ `data/database.json` หรือ `js/data.js` อีกต่อไป (ไฟล์ `js/data.js` จะถูกใช้เฉพาะตอนเชื่อมต่อ Apps Script ไม่ได้เท่านั้น)

ตัวกรอง (จังหวัด / ประเภทรถ EV / งานที่รับซ่อม) และ chip ด่วนจะถูกสร้างขึ้นอัตโนมัติจากข้อมูลใน JSON — ไม่ต้องแก้โค้ด `app.js`

## การเข้าถึงแบบแอดมิน (Admin-only)

การเพิ่ม/แก้ไขฐานข้อมูล (ปุ่ม "จัดการข้อมูล" มุมขวาบน) ถูกล็อกด้วย **ชื่อผู้ใช้ + รหัสผ่าน** ผู้ใช้ทั่วไปที่เข้ามาค้นหาอู่จะไม่เห็น/ไม่สามารถแก้ข้อมูลได้

- ค่าตั้งต้นอยู่ที่ค่าคงที่ `ADMIN_USERNAME` และ `ADMIN_PASSCODE` บนสุดของ `js/app.js` (ค่าเริ่มต้น: `admin` / `pk-admin-2026`) — **ควรเปลี่ยนก่อน deploy จริง**
- ต้องกรอกทั้งชื่อผู้ใช้และรหัสผ่านให้ตรงกันทั้งคู่ ถึงจะเข้าหน้าอัปโหลดได้
- หลัง login สำเร็จ ระบบจะจำสถานะไว้ใน `sessionStorage` จนกว่าจะปิดแท็บ หรือกด "ออกจากระบบแอดมิน" ในหน้าอัปโหลด

**⚠️ ข้อจำกัดด้านความปลอดภัยที่ควรทราบ:** เว็บนี้เป็น static site ไม่มี backend/เซิร์ฟเวอร์ตรวจสอบสิทธิ์จริง การกันด้วย username/password ฝั่ง JavaScript แบบนี้ป้องกันได้แค่ผู้ใช้ทั่วไปที่ไม่ได้ตั้งใจแกะโค้ด (deterrent) เท่านั้น — ค่าที่ตั้งไว้ยังอยู่ในไฟล์ `js/app.js` ซึ่งใครก็เปิดดูได้ผ่าน "View Page Source" ของเบราว์เซอร์ ถ้าต้องการความปลอดภัยที่แท้จริง (เช่น ป้องกันคนที่ตั้งใจเจาะระบบ) จำเป็นต้องมีระบบ login ฝั่งเซิร์ฟเวอร์ (backend/auth service) ซึ่งเกินขอบเขตของเว็บสถิตนี้

## การตั้งค่า Backend (Google Sheet ผ่าน Apps Script)

ฐานข้อมูลจริง (source of truth) ของเว็บนี้คือ **Google Sheet** ที่ผูกกับ **Google Apps Script Web App** (โค้ดอยู่ที่ `apps-script/Code.gs`) เว็บฝั่งหน้าบ้าน (`index.html` + `js/app.js`) เป็น static site เหมือนเดิม แต่โหลด/บันทึกข้อมูลผ่าน Web App นี้แทนไฟล์ JSON ในเครื่อง — โครงสร้างข้อมูล (schema) เหมือนเดิมทุกฟิลด์ ไม่เปลี่ยน แค่เปลี่ยนที่เก็บจริงจากไฟล์ในเครื่อง/GitHub มาเป็น Google Sheet

**วิธีตั้งค่า (ทำครั้งเดียว):**

1. เปิด [sheets.google.com](https://sheets.google.com) สร้าง Google Sheet ใหม่ (เปล่าๆ ก็ได้)
2. เมนู **Extensions (ส่วนขยาย) > Apps Script**
3. ลบโค้ดเริ่มต้นทั้งหมด แล้ววางเนื้อหาจากไฟล์ `apps-script/Code.gs` ที่ให้มาแทน
4. ที่แถบซ้ายของ Apps Script editor กด **Project Settings (รูปเฟือง)** > เลื่อนไป **Script Properties** > **Add script property**
   - Property: `API_KEY`
   - Value: ตั้งรหัสลับเอง (แนะนำสุ่มยาวๆ เช่นใช้ `openssl rand -hex 24` หรือ password generator) — **นี่คือกุญแจที่ใช้แทน GitHub token เดิม** เก็บอยู่ฝั่ง Apps Script เท่านั้น ไม่มีวันถูก commit ขึ้น git
5. กลับไปที่ตัวแก้ไขโค้ด เลือกฟังก์ชัน **`seedInitialData`** จาก dropdown ด้านบน แล้วกด **Run** (ครั้งแรกจะขึ้น popup ขอสิทธิ์เข้าถึง Google Sheet — กด "Advanced" > "Go to project (unsafe)" ได้ตามปกติ เพราะเป็นโปรเจกต์ของตัวเอง) ระบบจะสร้างชีตชื่อ "Garages" พร้อมข้อมูลตัวอย่าง 14 อู่ให้อัตโนมัติ
6. กดปุ่ม **Deploy > New deployment**
   - Select type: **Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
   - กด **Deploy** แล้วคัดลอก **Web app URL** ที่ได้ (ลงท้ายด้วย `/exec`)
7. เปิดไฟล์ `js/app.js` หาบรรทัด `const APPS_SCRIPT_URL = "https://script.google.com/macros/s/YOUR_DEPLOYMENT_ID/exec";` แล้วแทนที่ด้วย URL จริงที่ได้จากข้อ 6

**วิธีใช้ในเว็บ:** เข้าหน้า "จัดการข้อมูล (แอดมิน)" แล้ววาง `API_KEY` ที่ตั้งไว้ในข้อ 4 ลงในช่อง "API Key" ก่อนอัปโหลดไฟล์หรือกดรีเซ็ต ระบบจะไม่บันทึก key ลงไฟล์ใดๆ ทั้งสิ้น — เก็บไว้ใน `sessionStorage` ของเบราว์เซอร์เครื่องนั้นชั่วคราวเท่านั้น หายเองเมื่อปิดแท็บหรือกด "ออกจากระบบแอดมิน"

**ทำไมวิธีนี้ปลอดภัยกว่าการ commit ขึ้น GitHub โดยตรง:**
- `API_KEY` ตั้งและเก็บอยู่ฝั่ง Google Apps Script (Script Properties) เท่านั้น ไม่เคยอยู่ในไฟล์ที่ push ขึ้น GitHub เลยแม้แต่ครั้งเดียว จึงไม่มีความเสี่ยงเรื่องรั่วผ่าน git history เหมือนวิธี GitHub token เดิม
- Key นี้ให้สิทธิ์เขียนได้แค่ชีต "Garages" ชีตเดียวเท่านั้น ไม่ใช่ credential ที่เข้าถึงบัญชี GitHub/repo ทั้งหมด ต่อให้หลุดไปความเสียหายก็แคบกว่ามาก
- เปลี่ยน/หมุนเวียน key ใหม่ได้ทันทีจาก Script Properties โดยไม่กระทบส่วนอื่นของระบบเลย ไม่ต้องรอ deploy ใหม่
- ทุกครั้งที่แก้โค้ดใน `Code.gs` แล้วต้อง **Deploy > Manage deployments** > กดไอคอนดินสอ > **Version: New version** > Deploy ใหม่ ไม่งั้น Web app จะยังรันโค้ดเวอร์ชันเก่าอยู่

**ข้อควรระวัง:** ใครก็ตามที่นั่งอยู่หน้าเครื่อง/เบราว์เซอร์ที่แอดมิน login ค้างไว้ (เช่น เครื่องสาธารณะ) สามารถเปิด DevTools แล้วอ่าน key จาก sessionStorage ได้ — ควรกด "ออกจากระบบแอดมิน" ทุกครั้งหลังใช้งานเสร็จ โดยเฉพาะบนเครื่องที่ใช้ร่วมกับคนอื่น

## ฟีเจอร์

- ค้นหาแบบ real-time จากชื่ออู่ / จังหวัด / อำเภอ / ยี่ห้อรถ / งานที่รับซ่อม / ผู้ติดต่อ
- กรองตามจังหวัด, **อำเภอ/เขต**, ประเภทรถ EV, งานที่รับซ่อม (dropdown + chip ด่วน)
- เรียงผลลัพธ์ตามชื่ออู่ หรือจังหวัด
- ปุ่มลิงก์ด่วน: โทร / LINE / Facebook / Google Maps ต่อการ์ด
- **QR Code สำหรับติดหน้าร้าน** — แต่ละอู่มีปุ่ม "แสดง QR Code" เปิดโมดัลที่สลับได้ 2 แท็บ:
  - **QR แผนที่** — เข้ารหัสลิงก์ Google Maps ของอู่นั้น (ถ้าไม่มีลิงก์แผนที่ระบบจะสร้างลิงก์ค้นหาจากชื่อ+ที่อยู่ให้อัตโนมัติ)
  - **QR LINE** — ถ้าอัปโหลดลิงก์รูป QR Code LINE จริงไว้ในช่อง "QR Code Line" ระบบจะแสดงรูปนั้นตรงๆ (เป็น QR ทางการของร้าน) แต่ถ้าไม่มีจะสร้าง QR จากช่อง "Line" ให้อัตโนมัติแทน — ถ้าอู่ไหนไม่มีข้อมูล LINE เลย แท็บนี้จะถูกซ่อนไป
  
  ทั้งสองแบบมีปุ่มดาวน์โหลดเป็น PNG ไปพิมพ์ติดหน้าร้านได้เลย
- **อัปโหลดข้อมูลจาก Excel (เฉพาะแอดมิน)** — ปุ่ม "จัดการข้อมูล" มุมขวาบน ต้องกรอกชื่อผู้ใช้และรหัสผ่านก่อนถึงเปิด modal อัปโหลดได้ รองรับลาก-วางหรือเลือกไฟล์ `.xlsx` / `.xls` / `.csv` แล้วบันทึกขึ้น Google Sheet โดยตรง ให้ทุกคนเห็นข้อมูลเดียวกัน
- ธีมแผนภาพวงจร/สายไฟรถ EV บนพื้นหลังสีขาวสบายตา (accent เขียวโวลต์ + ฟ้า teal)
- Responsive ปรับสัดส่วนแยก 3 ระดับ: จอ desktop/laptop กว้าง (grid 3 คอลัมน์), laptop เล็ก/แท็บเล็ต (2 คอลัมน์), มือถือ (1 คอลัมน์ + ปุ่มขนาดใหญ่กดง่ายด้วยนิ้ว), รองรับคีย์บอร์ด focus และ `prefers-reduced-motion`
- Dependency ภายนอก: Google Fonts + [SheetJS](https://sheetjs.com) (โหลดจาก CDN เพื่ออ่านไฟล์ Excel/CSV ในเบราว์เซอร์ — ต้องมีอินเทอร์เน็ตตอนใช้ฟีเจอร์อัปโหลด) — deploy เป็น static site ได้ทันที (GitHub Pages เป็นต้น)

## การอัปโหลดข้อมูลผ่าน Excel (สำหรับแอดมิน — ไม่ต้องแก้โค้ด)

1. กดปุ่ม **"จัดการข้อมูล (แอดมิน)"** ที่หัวเว็บ แล้วกรอกชื่อผู้ใช้และรหัสผ่านแอดมิน
2. วาง **API Key** ลงในช่องด้านบนของ modal (ค่าเดียวกับที่ตั้งไว้ใน Apps Script Script Properties — ดูหัวข้อ "การตั้งค่า Backend" ด้านบน) — ต้องทำทุกครั้งที่เปิดแท็บใหม่ เพราะ key ไม่ถูกจำข้ามเซสชัน
3. กด **"ดาวน์โหลด Template Excel"** — ระบบจะสร้างไฟล์ `garage_template.xlsx` ให้ทันทีในเบราว์เซอร์ (ไม่ต้องพึ่งไฟล์แยก) มีหัวคอลัมน์ภาษาไทยตรงกับตารางต้นฉบับ พร้อมแถวตัวอย่าง 1 แถว และคำแนะนำในแถวบนสุด
4. กรอกข้อมูลอู่ในไฟล์ (1 แถว = 1 อู่) — ช่อง **ประเภทรถ EV** และ **งานที่รับซ่อม** คั่นหลายค่าด้วยจุลภาค `,` เช่น `BYD, NETA, MG`
5. ลากไฟล์มาวางในกล่อง หรือแตะเพื่อเลือกไฟล์
6. เลือกโหมด:
   - **เพิ่ม/อัปเดตต่อจากเดิม** — แถวที่มี "ลำดับ" ตรงกับของเดิมจะถูกอัปเดตทับ ส่วนแถวใหม่จะถูกเพิ่มเข้าไป (ค่าเริ่มต้น)
   - **แทนที่ข้อมูลทั้งหมด** — ล้างข้อมูลเดิมทั้งหมดแล้วใช้เฉพาะไฟล์ที่อัปโหลด
7. ระบบจะอ่านไฟล์ ผสานกับข้อมูลเดิมตามโหมดที่เลือก แล้วส่งไปเขียนทับชีต "Garages" บน Google Sheet ให้อัตโนมัติ พร้อมแจ้งจำนวนแถวที่นำเข้าสำเร็จและรายการแถวที่ข้าม (เช่น ไม่มีชื่ออู่หรือจังหวัด)

**หมายเหตุสำคัญ:**
- ข้อมูลที่อัปโหลด **จะถูกเขียนทับชีต "Garages" บน Google Sheet จริง** ทันทีที่กดอัปโหลดสำเร็จ ทุกเบราว์เซอร์/อุปกรณ์ที่เข้าเว็บจะเห็นข้อมูลชุดใหม่ในการโหลดหน้าเว็บครั้งถัดไปทันที (ไม่ต้องรอ build เหมือนตอนใช้ GitHub) — **ไม่สามารถย้อนกลับอัตโนมัติได้** ถ้าอัปโหลดผิดไฟล์ ให้เปิด Google Sheet โดยตรงแล้วดู **File > Version history** เพื่อกู้คืนเวอร์ชันก่อนหน้า หรืออัปโหลดไฟล์ที่ถูกต้องซ้ำอีกครั้ง
- ถ้าเบอร์โทรในไฟล์ Excel เลข 0 นำหน้าหายไป (ปัญหาคลาสสิกเวลาเก็บเป็นตัวเลขแทนข้อความ) ระบบจะพยายามเติม 0 คืนให้อัตโนมัติเมื่อเป็นไปได้ — แต่แนะนำให้ตั้ง format คอลัมน์โทรศัพท์เป็น **ข้อความ (Text)** ใน Excel ก่อนกรอกเพื่อความชัวร์ (Template ที่ดาวน์โหลดตั้งไว้ให้แล้ว)
- กด **"รีเซ็ตเป็นข้อมูลเริ่มต้น"** ในหน้าอัปโหลด เพื่อเขียนทับชีต "Garages" บน Google Sheet กลับเป็นชุดข้อมูลเริ่มต้นในไฟล์ `js/data.js` (ต้องยืนยันก่อนทุกครั้ง เพราะกระทบข้อมูลจริงบนเซิร์ฟเวอร์)
- เว็บ **ไม่มีการตรวจสอบล็อกการแก้ไขพร้อมกัน (concurrent edit)** — ถ้าแอดมิน 2 คนอัปโหลดพร้อมกันในเวลาไล่เลี่ยกัน คนที่กดสำเร็จทีหลังจะเขียนทับของคนแรก แนะนำให้แอดมินไม่อัปโหลดพร้อมกันหลายคนในเวลาเดียวกัน
